Contributing

Thanks for your interest in MiBox! Here are some ways you can get involved and help us improve MiBox. For the project itself, check out its Overview page.

Reporting Bugs

During downloading, deploying and employing the source, file a bug from these links listed below if you run across issues over all these projects produced by Miing.org and hosted on Github.com. Note that you have to apply for an account on Github.com to file them.

  • MiBox
  • MiGo

For these bugs from the upstream projects used in the MCI project, go over to their homepages to find out where to file them.

Contributing to Code

Code is King. We would like to get your code into our code base. So, please check out the source, pick up a bug or feature, and get coding accodring to fork a repo.

Coding Style Guidelines

As a rule of thumb, picking up the coding style that is shown inside source code is the best way to get the uniform and complete coding falvor and to avoid the coding style out of line between contributors.
